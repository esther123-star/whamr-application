1
Create your R2 bucket
Go to dash.cloudflare.com → R2 Object Storage → Create bucket. Name it whamr-media. Then go to Settings → Public Access → Allow Public Access and copy the pub-xxxx.r2.dev URL.
The bucket must be public for the memes to be accessible from the browser without authentication.
2
Create an API token
Go to R2 → Manage R2 API Tokens → Create API Token. Set permissions to Object Read & Write scoped to the whamr-media bucket. Copy the Access Key ID and Secret Access Key.
3
Fill in your credentials
Copy .env.r2.example → .env.r2 at the project root and fill in your values:
# Cloudflare R2 credentials
R2_ACCOUNT_ID=a1b2c3d4e5f6...
R2_ACCESS_KEY_ID=your_access_key
R2_SECRET_ACCESS_KEY=your_secret_key
R2_BUCKET_NAME=whamr-media
R2_PUBLIC_URL=https://pub-xxxx.r2.dev
MEDIA_SOURCE=C:\Users\...\Memes & Stickers
4
Install dependencies and upload
From the project root, install the upload script dependencies and run the upload:
cd scripts npm install node upload-to-r2.js
The script uploads 4 files in parallel, skips anything already on R2, retries failures 3 times, and shows live progress. Re-run any time you add new memes — it won't re-upload what's already there.
5
Regenerate the catalog
After the upload completes, generate a fresh data/memes.json with R2 URLs:
node generate-catalog.js
This reads the upload manifest, AI-categorises every file, deduplicates, and writes data/memes.json pointing at your R2 bucket. Then commit and push — the live site instantly serves real content.
