What Whamr does. A user opens Whamr, types "Nigerian side-eye" or "happy birthday" in plain English, gets exactly the right meme or sticker, and taps Share — straight to WhatsApp or Telegram, in under ten seconds. No scrolling through unrelated results. No signing up. No ads. Just the send.

Whamr is designed around one core action: the send. Every feature — search, browse, share, favorites, upload — exists to make that send faster and more satisfying. The platform sits at the intersection of Giphy's catalog depth and the Naija cultural intelligence that no Western meme platform has built properly.

The product is globally scoped. Naija content is the primary differentiator and anchor for the initial user base, not a regional limitation. The catalog, categories, and community tools all support content from any culture — the AI search understands slang, the community tabs let anyone add origin context, and the Sticker Pack Export works with WhatsApp packs from any region.

AI Features Specification

Four AI features power Whamr's intelligence layer. All four run client-side in the current prototype — no API calls, no backend. They are fast, free, and work offline.

A. Natural Language Search LIVE
Location memes.html + script.js (live) · 18-ai-search.html (mockup)
Entry function expandQuery() → builds synonym set for user's input
Scoring aiScore() runs fuzzy match — 1.0 exact · 0.9 prefix · 0.85 contains · 0.75 token prefix
Intent detection detectIntent() — regex patterns map queries to category hints
Synonym clusters 20+ clusters: funny → laugh/lol/hilarious, naija → nigeria/nollywood/lagos, sad → cry/tears/disappointed, dance → groove/moves/twerk, etc.
Categories detected dance, laughing, sad, naija, sports, reactions, awkward, love, birthday
Score threshold ≥ 0.3 to include in results; sorted descending
Known limitation Searches by title / filename — not visual content. Use Origin tab to add searchable descriptions for memes with unclear filenames.
Scale limit Works well under 10k items. At 10k+, migrate to Meilisearch or Algolia.
B. Auto-Categorisation LIVE
Location scripts/generate-catalog.js
Method 13 keyword rule arrays matched against filename → assigns category → extracts tags
Categories naija, dance, laughing, sad, reactions, awkward, sports, love, birthday, random, stickers
Tag extraction Filename split on delimiters (spaces, underscores, hyphens) → noise words stripped → max 8 tags per item
Deduplication Jaccard similarity on word sets, threshold 0.9 — removes near-duplicate filenames (519 removed from 9,128)
Output per item { id, title, type, category, tags, path, size }
Accuracy ~80% accurate at this scale using filename keywords. Good enough for prototype; improve with human review or vision API at scale.
C. Community Knowledge (Discussion · Origin · Variants) LIVE
Location memes.html modal · script.js communityKey() functions
Storage localStorage — keyed by meme ID, survives page refresh, local per device
Discussion tab Threaded comments: author name, text, timestamp, timeAgo() display ("2 hours ago")
Origin tab Fields: year, source platform, history text — editable by any visitor, no account required
Variants tab URL links to remixed or extended versions (YouTube, TikTok, Twitter) with title and timestamp
Purpose Crowdsource the metadata that can't be inferred from filenames — context, cultural meaning, history
Migration path When Clerk auth is live, replace localStorage with server-side storage keyed on memeId as foreign key
D. Duplicate Detection (Upload Guard) LIVE
Location lib/ai-search.ts · isDuplicate() function
Method Jaccard similarity — word-set intersection divided by union — on normalised titles
Threshold 0.85 similarity triggers duplicate warning
Output Returns the most similar existing item — shown as a warning card before the user confirms upload
User action User can dismiss warning and upload anyway (e.g. deliberate variant) or cancel

Community Features

Whamr's community layer turns a search tool into a place. Every meme has a public space for discussion, history, and remixes. Dojos give users themed hubs to gather around shared meme cultures.

Dojos (10-dojos.html)

Dojos are community groups built around themes. Examples: Naija Meme HQ, Football Meme FC, Afrobeats Dance Vault. Each Dojo has a member count, an activity feed, meme battle threads, and a rules panel. Any user can create a Dojo or join an existing one.

Discussion Tabs

Each meme has a Discussion tab in its detail modal. Users post comments with a display name they type inline — no account required. Comments show timestamp and a timeAgo() display. All data lives in localStorage under the meme's ID key.

Origin Tabs

The Origin tab lets anyone record the meme's history: year it appeared, source platform (Twitter, TikTok, Nollywood, etc.), and a free-text narrative. This is the mechanism for adding searchable context to memes whose filenames give no clues. It's also where Naija meme provenance gets documented — which movie scene, which musician, which viral moment.

Variants Tabs

Variants are external links to remixed or extended versions of the meme — the original YouTube clip, a TikTok remix, an extended Twitter thread. Users paste a URL and a title. This builds a web of related content without hosting it all.

Direct Messages

The DM screen (09-messages.html) shows a conversation list on the left and a thread on the right. The thread has an inline sticker picker — users can search and insert a meme directly into the message without leaving the chat. This is the core send-inside-Whamr use case.

Notifications

The Notifications screen (08-notifications.html) shows an activity feed tabbed by All, Likes, Comments, and Follows. Notifications link back to the relevant meme or profile.

Whamr's Edge

Five things Whamr does that no other platform does together: AI natural language search, community knowledge (Discussion + Origin + Variants tabs), a dedicated Naija meme library, sticker pack export for WhatsApp/Telegram/Signal/Discord, and upload with no account required. Any one of these would be a differentiator. Having all five is the product bet.

Business Model

Free Tier (Current)

Browse, save, share, and download: free, no account, no ads. Upload: free, stored locally (IndexedDB) or to Cloudinary. This is the current state — the full product experience is free with no paywall.

Creator Tier (Planned)

  • Persistent uploads — files stored server-side, not lost when localStorage clears
  • Analytics dashboard — real view, share, and follower data tied to their uploads
  • Creator profile — public page with follower system, meme grid, bio
  • Monetisation tools — donations, tipping, or pay-to-send branded packs

API Tier (Planned)

  • REST API access to the meme catalog — search, browse, fetch by ID or category
  • Free tier: 100 requests/day — for developers building bots or integrations
  • Pro tier: higher rate limits for production apps (pricing TBD)

Voice Notes

Whamr is Naija-aware but globally readable. Use pidgin references where they land naturally — but never as performance. The product speaks to a 19-year-old in Lagos and a 25-year-old in East London with the same directness. Naija culture is the flavour, not the fence.

Numbers give confidence. Prefer "413 memes" over "hundreds of memes." Prefer "Share in one tap" over "Share quickly." Specificity is respect for the reader.

Decision Log

Key architectural and product decisions made during development — recorded here so the team understands not just what was built, but why.

01
Static HTML mockups over framework prototypes

All 19 screens are plain HTML/CSS/JS — no Node server, no bundler. Anyone can open them in a browser from a file path. Faster iteration for design review, easier to share with stakeholders who are not technical, and zero environment setup friction.

02
Cloudinary → Cloudflare R2 migration planned

Cloudinary's free tier caps egress at 25 GB/month. Cloudflare R2 has zero egress fees. The entire 9,128-file library (~9 GB) fits in R2's 10 GB free tier. The upload script is written and ready — this migration is the single highest-leverage task to unlock the full catalog.

03
AI search in script.js, not a backend API

Prototype speed. Client-side synonym expansion and fuzzy scoring works well for 413 items and is still usable at 8,609. No API latency, no server cost. At 10k+ items the in-memory scan becomes slow — migrate to Meilisearch or Algolia then.

04
localStorage for community data

No backend yet. localStorage gives instant persistence without a database. The data structure (keyed by meme ID) is designed to migrate directly to server-side PostgreSQL rows when Clerk auth goes live. The migration path is deliberate.

05
Catalog generated from filenames, not visual analysis

Extracting meaningful tags from video frames requires expensive ML infrastructure (a vision model, GPU time, API costs). Filename keyword matching gives ~80% accuracy for categorisation at this scale for free. Good enough for prototype — improve with human review or a vision API when the catalog is live and the audience is real.

06
19 mockup screens — covering all core user journeys

The screen list was designed to cover every user journey: discover → view → share → create → upload → community → admin → mobile. 19 screens is comprehensive without being excessive. Each screen is designed to stand alone so stakeholders can review individual journeys without needing the full flow.

07
Naija content as primary differentiator

No major meme platform prioritises Nigerian or African content. Giphy and Tenor have minimal Naija coverage — largely what gets uploaded by individuals. Whamr has a dedicated Naija category, Naija-aware synonym expansion in the AI search, and a library built around real Nigerian cultural moments. This is the defensible position no Western platform can replicate quickly.

08
whamr.com domain taken — running on Vercel subdomain

The current whamr-application.vercel.app URL is fine for prototype and stakeholder review but not for any public launch. Domain decision should happen before Phase 1 ships. Top candidates are jollify.com (globally brandable) and whamz.com (brand continuity).

09
Dual design system — dark theme on live site, warm theme on mockups

The live Next.js site uses a dark theme (#0a0a0f) that matches Clerk's dark modal aesthetic. The static HTML mockups use the warm sunset palette (#FFFAF5 + #FF4500). Both are intentional. The dark theme suits signed-in product screens. The warm theme suits discovery, marketing, and design review.

10
IntersectionObserver for infinite scroll, no pagination library

No external dependency, no bundle weight, works in all modern browsers. 24 items per page keeps initial load fast. The sentinel div is placed 300px before the viewport edge — this early trigger means the next batch loads before the user reaches the bottom, so scroll feels instant. Simple, fast, and zero maintenance.
